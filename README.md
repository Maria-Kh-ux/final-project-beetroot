# 🎓 Фінальний проєкт: Fridge Chef (Розумний кулінарний помічник)

Ласкаво просимо до **Fridge Chef** — сучасного, швидкого та інтуїтивно зрозумілого вебдодатка на React, створеного як випускний проєкт курсу з Frontend-розробки. 

Цей додаток допомагає раціонально використовувати продукти, що вже є в холодильнику, мінімізувати харчові відходи, вести зручний список покупок для страв та отримувати персоналізовані рекомендації страв на основі ваших дієт і кулінарної історії. Додаток повністю підтримує **офлайн-режим (Offline-First)** і спроєктований з фокусом на екстремальну швидкість завантаження.

---

## 🚀 Основні можливості проєкту

1. **Інтерактивний холодильник (Fridge Inventory)**
   - Зручний вибір наявних інгредієнтів по категоріях (Молочні продукти, Овочі, Бакалія, Фрукти тощо).
   - Можливість моментально додавати власні унікальні інгредієнти, яких немає у списку популярних.
   - Очищення холодильника в один клік та динамічний підрахунок кількості наявних продуктів.

2. **Автоматична фільтрація страв за складністю**
   - **Легкі та швидкі страви (2-4 інгредієнти)**: швидкі сніданки чи перекуси з мінімальним часом підготовки (наприклад, тости, швидка яєчня).
   - **Середні страви (4-7 інгредієнтів)**: помірні за складом та часом приготування (наприклад, паста Карбонара, салат Капрезе, крем-суп).
   - **Складні страви (7+ інгредієнтів)**: повноцінні кулінарні шедеври для родинних обідів (наприклад, домашній плов, запечена курка).

3. **Локальна інтелектуальна система рекомендацій (Personalized Recommendation Engine)**
   - Аналізує ваші кулінарні дії та нараховує динамічні бали для кожної страви:
     - **+45 балів** за високий відсоток збігу інгредієнтів у вашому холодильнику.
     - **+25 балів**, якщо категорія страви додана до ваших "Улюблених".
     - **до +30 балів** за історію приготування (чим частіше ви готуєте страву, тим більше система пропонує схожі або її саму).
   - Жорсткі фільтри за типом дієти (Усе, Вегетаріанська, Веганська, Безглютенова, Низьковуглеводна) та за часовим лімітом приготування (5-60 хвилин).

4. **Інтерактивний список покупок (Shopping List Manager)**
   - Можливість автоматично додавати відсутні продукти для будь-якої обраної страви за один клік (кнопка "Купити відсутні").
   - Додавання довільних домогосподарських товарів вручну з зазначенням ваги чи об'єму.
   - Інтерактивне викреслювання придбаного, очищення купленого та повне видалення списку.

5. **Офлайн-режим та локальна синхронізація (Offline-First)**
   - Усі дані (вміст холодильника, список покупок, дієти, історія приготувань, темна/світла теми) автоматично та миттєво зберігаються у `localStorage`.
   - Вбудована система відстеження підключення до мережі (`navigator.onLine`). У разі зникнення Інтернету додаток виводить делікатну плашку-сповіщення, підтверджуючи повну автономну працездатність.

6. **Оптимізація швидкості (PageSpeed Insights >= 85)**
   - **Чистий React 19** без важких залежностей.
   - Оптимізована робота з анімаціями завдяки легким переходам CSS та `motion`.
   - Інлайнові та оптимізовані SVG-іконки з бібліотеки `lucide-react`.
   - Завантаження лише високоякісних сучасних шрифтів `Inter` та `Space Grotesk` через оптимізовані Google Fonts API.

7. **Сучасний Адаптивний UI & Подвійна тема (Light/Dark Mode)**
   - Бездоганний вигляд на мобільних телефонах, планшетах та десктопах завдяки мобільно-орієнтованому Tailwind CSS v4.
   - Зручна нижня навігаційна панель для мобільних пристроїв та повноцінна шапка для широких екранів.

## 🛠️ Технологічний стек

* **Бібліотека**: React 19 (Functional Components, Hooks: `useState`, `useEffect`, `useMemo`).
* **Збирач проєкту**: Vite 6 (надшвидкий гарячий запуск та оптимальне стиснення коду).
* **Стилізація**: Tailwind CSS v4 (використання CSS-змінних нової версії, `@theme` директив, вбудованих утиліт темної теми).
* **Анімації**: `motion` для плавних мікро-взаємодій, тостів та модальних вікон.
* **Іконки**: `lucide-react` (модульні векторні SVG).

## ⏱️ Керівництво з налаштування середовища (4-6 хв)

Дотримуйтесь цих покрокових кроків для запуску проєкту на власному комп'ютері та подальшої роботи з кодом.

### Крок 1. Передумови (Prerequisites)
Переконайтеся, що на вашому комп'ютері встановлено середовище виконання **Node.js** версії **18.0.0** або новішої та менеджер пакетів **npm** (входить до складу інсталятора Node.js).
* Перевірити версію Node: `node -v`
* Перевірити версію NPM: `npm -v`

### Крок 2. Розархівування та перехід у каталог
Якщо ви завантажили ZIP-архів проєкту з AI Studio:
1. Розархівуйте його у зручне місце.
2. Відкрийте термінал або командний рядок та перейдіть до папки проєкту:
   ```bash
   cd шлях_до_папки_з_проєктом
   ```

### Крок 3. Встановлення залежностей
Встановіть усі бібліотеки, прописані в файлі `package.json`:
```bash
npm install
```
*Цей крок оновить та завантажить локальну папку `node_modules` для підтримки React, Tailwind, Motion та Lucide.*

### Крок 4. Запуск у режимі розробки (Development)
Для локального тестування та написання коду запустіть вбудований dev-сервер:
```bash
npm run dev
```
Після успішного запуску в консолі з'явиться посилання (зазвичай `http://localhost:3000`). Відкрийте його у браузері. Будь-які зміни в коді будуть миттєво відображатися на екрані без перезавантаження сторінки!

### Крок 5. Перевірка лінтера
Перед збиранням перевірте проєкт на відсутність помилок типізації TypeScript:
```bash
npm run lint
```
*Лінтер автоматично підтвердить сумісність кодів з суворими стандартами типізації.*

## 🌐 Посібник з розгортання проєкту на сервері (Production Deployment)

Коли додаток готовий для демонстрації викладачам курсу чи користувачам, виконайте фінальну оптимізацію та деплой.

### 1. Компіляція та збірка (Build)
Запустіть процес створення оптимізованого виробничого бандлу:
```bash
npm run build
```
Цей процес скомпілює ваш TypeScript код у чистий JavaScript, видалить зайві відлагоджувальні коментарі та максимально стисне CSS. Результат збірки буде збережено у папці `/dist`.
*Ця папка містить статичні `index.html`, `js` та `css` файли, які готові для розміщення на будь-якому сервері.*

### 2. Завантаження проєкту на GitHub
Створіть репозиторій на GitHub та виконайте наступні команди у терміналі проєкту:
```bash
# Ініціалізувати локальний Git-репозиторій
git init

# Додати всі файли до індексу
git add .

# Створити перший коміт
git commit -m "feat: Fridge Chef graduation project initial release"

# Налаштувати гілку та зв'язати з віддаленим репозиторієм на GitHub
git branch -M main
git remote add origin https://github.com/ваше_ім'я_користувача/ім'я_репозиторію.git

# Відправити код на GitHub
git push -u origin main
```

### 3. Безкоштовне розгортання (Hosting Options)

Оскільки додаток є клієнтським SPA, ви можете розгорнути його за кілька хвилин абсолютно безкоштовно:

#### Варіант А. Розгортання через GitHub Pages (Найпопулярніше)
1. Встановіть пакет gh-pages: `npm install gh-pages --save-dev`
2. Додайте у `package.json` поле `"homepage": "https://ваше_ім'я.github.io/назва_репозиторію"`
3. Додайте скрипти у розділ `"scripts"` у `package.json`:
   - `"predeploy": "npm run build"`
   - `"deploy": "gh-pages -d dist"`
4. Виконайте команду в терміналі: `npm run deploy`
5. Ваш проєкт буде опубліковано в мережі Інтернет!

#### Варіант Б. Розгортання на Vercel або Netlify (Найпростіше)
1. Перейдіть на сайт [Vercel](https://vercel.com/) або [Netlify](https://www.netlify.com/).
2. Авторизуйтесь за допомогою свого акаунту GitHub.
3. Натисніть **"Add New Project"** (Додати новий проєкт) та оберіть ваш репозиторій.
4. Платформа автоматично розпізнає Vite-проєкт та налаштує команди збірки (`npm run build`) і вихідний каталог (`dist`).
5. Натисніть **"Deploy"**. Через 30 секунд ваш додаток отримає публічну URL-адресу з безкоштовним SSL-сертифікатом безпеки!

## 👩‍🎓 Висновок випускного проєкту

**Fridge Chef** успішно демонструє практичне застосування сучасних frontend-патернів:
* Робота з багаторівневим локальним станом у React.
* Реалізація інтелектуальної фільтрації на клієнті без затримок мережі.
* Повна стійкість додатка до перебоїв з інтернетом (Offline-First).
* Відповідність суворим показникам PageSpeed за рахунок чистих рішень без навантаження на систему.
